Expectation 'Paper'

Why a website instead of a paper?

Firstly, I wanted to visualize my learning journey. I felt that a combination of the GROW and I-WE-IT frameworks would best reflect my path. The order—Reality, Obstacles, I, Way Forward, We, It, and Goals—represents how I learn: starting from where I am now and the challenges I face, moving through action and collaboration, toward short-term ("It") and finally long-term goals. I created this mountain graphic in Photoshop to illustrate the metaphor. Each step can be explored by clicking the corresponding word.

Secondly, the website allows for integrated feedback and tracking. In the bottom left corner, the button ‘Help me track progress’ lets my teammates share input throughout the project. I will link this to a live results section next week, so that not only I, but also others can track my progress.

Finally, this project gave me a chance to apply and develop my web skills. I created this website using HTML in Visual Studio Code, with the Live Preview extension. I used ChatGPT for fast coding. GitHub and GitHub Pages were used to build and host the site. The website runs on HTML, CSS (with Flexbox for layout), and vanilla JavaScript to handle overlays and interactivity. Formspree is used to collect feedback via the tracker form and send it directly to my email, allowing the site to function without a backend.

I hope this gives a clear and engaging view of my learning process.
